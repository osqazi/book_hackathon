# تصدیق کی تنصیب

یہ دستاویز انسان نما روبوٹکس کتاب پروجیکٹ میں تصدیق کے نظام کو سیٹ اپ اور استعمال کرنے کا طریقہ بیان کرتی ہے۔

## جائزہ

تصدیق کا نظام [Better-Auth](https://better-auth.com/) استعمال کرتے ہوئے بنایا گیا ہے، جو TypeScript کے لیے ایک جامع تصدیق فریم ورک ہے۔ یہ صارف کی مہارت کی بنیاد پر مواد کو ذاتی بنانے کے لیے اضافی پس منظر پروفائلنگ کے ساتھ ای میل/پاس ورڈ تصدیق فراہم کرتا ہے۔

## خصوصیات

- **ای میل/پاس ورڈ تصدیق**: ای میل اور پاس ورڈ کے ساتھ محفوظ سائن اپ اور سائن ان
- **پس منظر پروفائلنگ**: صارف کے سافٹ ویئر اور ہارڈ ویئر کے تجربے کے بارے میں معلومات جمع کریں
- **مواد کی ذاتی کاری**: صارف کے پس منظر کی بنیاد پر مواد کو ڈھالیں
- **مستقل سیشن**: ڈیفالٹ کے طور پر 7 دن کی سیشن مستقل مزاجی
- **محفوظ اسٹوریج**: صارف ڈیٹا انکرپشن کے ساتھ محفوظ طریقے سے محفوظ کیا جاتا ہے

## فن تعمیر

تصدیق کا نظام پر مشتمل ہے:

1. **Auth Server**: Better-Auth چلانے والا ایک علیحدہ Node.js سرور (port 3001 پر)
2. **Frontend Components**: سائن اپ، سائن ان، اور پروفائل مینجمنٹ کے لیے React کمپوننٹس
3. **Docusaurus Integration**: پلگ ان جو ایپ کو تصدیق کے سیاق و سباق کے ساتھ لپیٹتا ہے
4. **Personalization Engine**: کمپوننٹس جو صارف کے پس منظر کی بنیاد پر مواد کو ڈھالتے ہیں

## سیٹ اپ

### پیشگی تقاضے

- Node.js 18+ انسٹال شدہ
- Docusaurus پروجیکٹ سیٹ اپ
- SQLite (development) یا PostgreSQL (production) ڈیٹا بیس

### تنصیب

تصدیق کی انحصار پروجیکٹ کے حصے کے طور پر پہلے سے انسٹال ہیں:

```bash
npm install better-auth drizzle-orm drizzle-kit better-sqlite3
```

### ترتیب

1. **Environment Variables**:
   اپنی `.env` فائل کو تصدیق کی ترتیبات کے ساتھ بنائیں یا اپ ڈیٹ کریں:

   ```
   AUTH_URL=http://localhost:3001
   ```

2. **Auth Server چلائیں**:
   ```bash
   cd auth-server
   npm install
   npm run dev
   ```

3. **Docusaurus Server شروع کریں**:
   ```bash
   npm start
   ```

## استعمال

### سائن اپ

1. سائن اپ صفحہ پر جائیں
2. اپنا ای میل اور پاس ورڈ درج کریں (کم از کم 8 حروف)
3. اپنے سافٹ ویئر اور ہارڈ ویئر کے تجربے کے بارے میں پس منظر کے سوالات کے جوابات دیں
4. آپ کی پروفائل مواد کو ذاتی بنانے کے لیے استعمال کی جائے گی

### سائن ان

1. سائن ان صفحہ پر جائیں
2. اپنا ای میل اور پاس ورڈ درج کریں
3. آپ کی تصدیق ہو جائے گی اور مواد آپ کی پروفائل کی بنیاد پر ذاتی بنایا جائے گا

### ذاتی مواد

`ContentAdapter` کمپوننٹ خودکار طور پر آپ کے پس منظر کی معلومات کی بنیاد پر مواد کو ڈھالے گا۔

## کمپوننٹس

### SignupForm

`SignupForm` کمپوننٹ پس منظر پروفائلنگ سوالات کے ساتھ رجسٹریشن انٹرفیس فراہم کرتا ہے۔

### SigninForm

`SigninForm` کمپوننٹ لاگ ان انٹرفیس فراہم کرتا ہے۔

### ContentAdapter

`ContentAdapter` کمپوننٹ مواد کو لپیٹتا ہے اور صارف کے پس منظر کی بنیاد پر اسے ڈھالتا ہے۔

## API Endpoints

Auth سرور درج ذیل endpoints فراہم کرتا ہے (http://localhost:3001 کے مطابق):

- `POST /api/auth/sign-up` - نیا صارف رجسٹر کریں
- `POST /api/auth/sign-in` - صارف کی تصدیق کریں
- `POST /api/auth/sign-out` - صارف کو لاگ آؤٹ کریں
- `GET /api/auth/get-session` - موجودہ سیشن حاصل کریں

## پس منظر کے سوالات

نظام کثیر انتخابی سوالات کے ذریعے آپ کے تجربے کے بارے میں معلومات جمع کرتا ہے:

### سافٹ ویئر کا پس منظر
- پروگرامنگ زبانوں سے واقفیت
- فریم ورکس/لائبریریز کا تجربہ
- سافٹ ویئر ترقی کے تجربے کی سطح

### ہارڈ ویئر کا پس منظر
- روبوٹکس پلیٹ فارمز کا تجربہ
- پروجیکٹس میں استعمال شدہ سینسرز
- کام کیے گئے Actuators
- ہارڈ ویئر ترقی کے تجربے کی سطح

## سیکیورٹی

- پاس ورڈز صنعتی معیار کے الگورتھم استعمال کرتے ہوئے ہیش کیے جاتے ہیں
- سیشن محفوظ ہیں اور 7 دن کی میعاد ختم ہو جاتی ہے
- پروڈکشن میں HTTPS ضروری ہے
- ذاتی کاری کا ڈیٹا الگ اور محفوظ طریقے سے محفوظ کیا جاتا ہے

## ٹربل شوٹنگ

### Auth Server جواب نہیں دے رہا

- یقینی بنائیں کہ auth سرور port 3001 پر چل رہا ہے
- کسی بھی غلطی کے پیغامات کے لیے کنسول چیک کریں
- Docusaurus اور auth سرور کے درمیان نیٹ ورک کنیکٹیویٹی کی تصدیق کریں

### مواد ذاتی نہیں ہو رہا

- یقینی بنائیں کہ صارف مناسب طریقے سے تصدیق شدہ ہے
- تصدیق کریں کہ پس منظر کی معلومات فراہم کی گئی ہے
- کسی بھی غلطی کے لیے براؤزر کنسول چیک کریں

## ماحولیاتی ترتیب

پروڈکشن میں، یقینی بنائیں کہ:

- تمام تصدیق endpoints کے لیے HTTPS استعمال کریں
- مناسب سیشن اسٹوریج (Redis، وغیرہ) ترتیب دیں
- مناسب لاگنگ اور مانیٹرنگ سیٹ اپ کریں
- auth endpoints کے لیے rate limiting نافذ کریں
